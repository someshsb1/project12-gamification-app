Functional Description Currently, student performance on GBI Case Studies is recorded and displayed in a Fiori app, Global Bike Monitor, that enables the professors to see student performance on each step for a specified Case Study. This functional requirement is to make the information in the GBI monitor available in JSON format through an API to be consumed by another application, Gamified GBI Web App. Gamification Logo => Breakage fixed, Redirects to Welcome Page. Chat Feature Overview of the Chat Feature The Gamification application now includes a real-time chat feature, implemented using the ASP.NET SignalR library. This feature enhances user interaction by allowing instant communication, akin to other asynchronous chat applications. Technical Implementation SignalR Integration: Utilizes SignalR for real-time, bidirectional communication between server and clients. User Retrieval: Retrieves all users for chat participation, supporting direct user-to-user interactions. Asynchronous Communication: Operates asynchronously, enabling real-time messaging without page refreshes. User Interface Chat Window: Features a chat window where real-time messages are displayed. Real-time Interaction: Messages appear instantly, facilitating dynamic conversations. Notifications: Informs users of new messages, keeping them updated. Unread Message Indicator: Includes a label showing the number of unread messages, enhancing user awareness. Message Sorting: Chats are sorted based on recent messages, ensuring the most active conversations are easily accessible. Future Enhancements Group Chats: Supports creating and participating in group chats. File Sharing: Allows file sharing within chat sessions.
